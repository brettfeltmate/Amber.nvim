# Changelog

All notable changes to the amber.nvim project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Initial release of amber.nvim colorscheme
- Complete standalone plugin architecture
- Self-contained color utilities (no external dependencies)
- Comprehensive plugin integrations:
  - TreeSitter syntax highlighting
  - Lazy.nvim plugin manager
  - Blink.cmp completion
  - Snacks.nvim UI components
  - WhichKey keybinding helper
  - NvimTree file explorer
  - FzfLua fuzzy finder
  - GitSigns git integration
- Extensive configuration system with:
  - Style customization (italic, bold, etc.)
  - Transparency options
  - Sidebar and float styling
  - Color and highlight overrides
- Terminal color support
- Modern plugin architecture following best practices

### Features
- 🎨 Beautiful amber-inspired color palette
- ⚡ Performance optimized loading
- 🔧 Highly configurable
- 🔌 Extensive plugin support
- 📚 Comprehensive documentation

## [1.0.0] - Initial Release

### Added
- Core amber colorscheme implementation
- Plugin manager compatibility (Lazy.nvim, Packer.nvim)
- Basic documentation and README

## Development Notes

This changelog will be updated as new features, fixes, and versions are released. The amber.nvim colorscheme was designed to provide a warm, elegant alternative to existing Neovim themes while maintaining modern plugin architecture standards.
